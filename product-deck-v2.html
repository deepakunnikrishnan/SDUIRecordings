The Problem

Shipping UI changes takes weeks. It shouldn't.

Every card layout change, every new toggle, every re-ordered section requires a full mobile release. QA, store review, phased rollout — just to move a button.

And every bank client ships the same layout, because customisation costs another release cycle.

The Principle

Move decisions from the build to the server.

The server describes what to show, whether to show it, and with what data. The client knows only how to render it. UI changes become configuration changes — deployed in seconds, not weeks.

One binary. Every bank sees a different layout, different data, different brand.

Three Ways to Embed

The SDK fits wherever the bank needs it.

Mode 1
Full Screen
Launch CardManagement as a standalone screen. Bank picks its own entry point, colors, and layout.
Mode 2
Tab / Screen
Embed CardManagement as a tab inside the host app. Host controls the nav; SDK controls the content.
Mode 3
Widget
Drop a single card slot anywhere in the host UI — a balance card on the home screen, no extra screens needed.
Three Banks. One APK.

Every bank gets its own layout, brand, and data — from the server.

  • NovaPay — indigo gradient card, lock toggle, transaction list, 3 sections
  • Apex Bank — green theme, account banner, activity feed, no lock control
  • Meridian — purple theme, portfolio header, card identity, 4 sections

No conditional code. No feature flags. Each bank sends an X-Bank-Id header — the server resolves a different manifest and returns different data.

Live Content Management

Change a layout without touching the app.

  • Edit the server manifest — add a slot, rename a label, reorder sections
  • Save the file — no build, no deploy, no store review
  • Pull to refresh in any of the three modes — new layout appears instantly
  • Works identically in Activity, Screen, and Widget mode — same server, same loop
What This Means for a Bank Client

Your PM can ship a layout change on a Tuesday afternoon.

  • No SDK release gating the change — edit the manifest, it's live
  • Embed card features anywhere in your app — home screen widget, deep-link target, tab
  • Full brand control — gradient colors, chip, toggle, fonts all configured per bank
  • A/B test layouts across user segments with zero client-side code
